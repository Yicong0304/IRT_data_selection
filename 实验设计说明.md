# 基于 Pairwise IRT 的 LLM 评测数据筛选实验

## 一、研究动机

在 LLM-as-a-Judge 范式下，我们用多个小模型（judge）对待评测模型的回答做成对比较。但并非所有评测题目（prompt）和评审者（judge/rater）都同等有效——有些题目区分不了模型好坏，有些 judge 打分接近随机。

核心问题：**能否利用 IRT（项目反应理论）估计出的参数，自动筛选出高质量的 prompt 和 rater，用更少的数据达到甚至超过全量数据的评测效果？**

---

## 二、实验 Pipeline 总览

```
Step 1: run_rollout.py    →  4 个待评测模型对 100 道 prompt 生成回答
Step 2: run_judge.py      →  3 个 judge 模型对所有模型配对做成对比较
Step 3: run_irt.py        →  拟合 Pairwise IRT 模型，估计所有参数
Step 4: run_filter_experiment_v2.py  →  利用 IRT 参数做筛选实验
```

---

## 三、各阶段详细说明

### Step 1: 模型回答生成（run_rollout.py）

**待评测模型（4 个，均为 13B 级别）：**

| 模型名 | 本地路径 |
|--------|---------|
| vicuna-13b | vicuna-13b-v1.3 |
| wizardlm-13b | WizardLM-13B-V1.2 |
| koala-13b | koala-13B-HF |
| alpaca-13b | alpaca-13b |

**评测题目：** 100 道 prompt（来自 `irt_experiment_prompts.json`）

**推理方式：** 使用 vLLM 加速推理，每个模型套用各自的对话模板（Vicuna/WizardLM 用 `USER:/ASSISTANT:` 格式，Alpaca 用 `### Instruction/### Response` 格式，Koala 用 `BEGINNING OF CONVERSATION` 格式）。

**输出：** 每个模型一个 JSON 文件，存于 `Rollout/` 目录。

---

### Step 2: LLM-as-a-Judge 成对比较（run_judge.py）

**Judge 模型（3 个 7B 级别小模型）：**

| Judge 名 | 模型 | 对应 IRT 中的角色 |
|----------|------|------------------|
| qwen | Qwen2.5-7B-Instruct | Rater 1 |
| mistral | Mistral-7B-Instruct-v0.3 | Rater 2 |
| internlm | InternLM2.5-7B-Chat | Rater 3 |

**比较方式：** 对 4 个模型的所有两两配对（C(4,2)=6 对），每道 prompt、每个 judge 都做一次比较，judge 输出 "A" 或 "B" 表示哪个回答更好。

**数据量：** 100 prompt × 6 模型对 × 3 judge = 1800 条比较记录

**输出：** `judge_results.json`，每条记录包含 `model_a, model_b, outcome, judge, prompt_id`。

### Step 3: Pairwise IRT 模型拟合（run_irt.py + irt_raw_code.py）

#### 3.1 模型公式

$$P(A > B \mid r, p) = \lambda_p \cdot \sigma\!\Big(\alpha_r \cdot \alpha_p \cdot (\theta_A - \theta_B) - \beta_r - \gamma_p\Big) + \frac{1 - \lambda_p}{2}$$

其中 $\sigma(\cdot)$ 为 sigmoid 函数。

#### 3.2 参数说明与对应关系

| 符号 | 名称 | 维度 | 含义 | 先验分布 |
|------|------|------|------|---------|
| $\theta_i$ | Model Ability | 每个待评测模型 1 个（共 4 个） | 模型的综合能力值，越高越强 | $\mathcal{N}(0, 1)$ |
| $\alpha_r$ | Rater Discriminability | 每个 judge 1 个（共 3 个） | judge 区分模型好坏的能力。$\alpha_r > 1$ 表示该 judge 比平均水平更敏锐，$\alpha_r < 1$ 表示较迟钝 | $\log\alpha_r \sim \mathcal{N}(0, 0.5)$ |
| $\beta_r$ | Rater Bias | 每个 judge 1 个（共 3 个） | judge 的系统性偏差。$\beta_r > 0$ 表示倾向于选 B，$\beta_r < 0$ 倾向于选 A | $\mathcal{N}(0, 1)$ |
| $\alpha_p$ | Prompt Discriminability | 每道 prompt 1 个（共 100 个） | 该 prompt 区分模型能力的有效程度。高 $\alpha_p$ 的题目能放大能力差异 | $\log\alpha_p \sim \mathcal{N}(0, 0.5)$ |
| $\gamma_p$ | Prompt Difficulty Offset | 每道 prompt 1 个（共 100 个） | 该 prompt 的固有偏移。$|\gamma_p|$ 大表示不管模型能力如何，结果都倾向某一方 | $\mathcal{N}(0, 1)$ |
| $\lambda_p$ | Prompt Feasibility | 每道 prompt 1 个（共 100 个） | 该 prompt 的可行性/信息量。$\lambda_p = 1$ 表示结果完全反映能力差异，$\lambda_p \to 0$ 表示结果接近随机猜测 | $\text{Beta}(1, 1)$ |

#### 3.3 公式直觉解读

1. **核心：** $\theta_A - \theta_B$ 是两个模型的能力差，这是决定胜负的根本因素
2. **放大/缩小：** $\alpha_r \cdot \alpha_p$ 联合控制 sigmoid 的陡峭程度——好的 judge + 好的 prompt 让比较结果对能力差异更敏感
3. **偏移：** $\beta_r + \gamma_p$ 把 sigmoid 的中心点偏移——即使两个模型能力相同，偏差也可能导致非 50/50 的结果
4. **噪声衰减：** $\lambda_p$ 控制信号与噪声的混合比例——低可行性的 prompt 让结果退化为随机

#### 3.4 推断方法

使用 Pyro 框架的 SVI（Stochastic Variational Inference）进行变分推断：
- 优化器：Adam (lr=0.03)
- 损失函数：Trace_ELBO
- 训练轮数：3000 epochs
- 变分族：均值场（Mean-field），每个参数独立的正态/Beta 分布

### Step 4: IRT 参数驱动的数据筛选实验（run_filter_experiment_v2.py）

#### 4.1 实验目标

验证 IRT 估计出的参数能否指导我们：
1. **筛 prompt：** 用更少的高质量题目保持甚至提升评测效果
2. **筛 rater：** 去掉低质量 judge 的数据减少噪声
3. **联合筛选：** 两个维度同时筛选的叠加效果

#### 4.2 Part 1：Prompt 筛选

**Prompt 质量评分公式：**

$$\text{quality\_score}_p = \frac{\alpha_p \cdot \lambda_p}{1 + |\gamma_p|}$$

设计逻辑（直接从 IRT 公式推导）：
- $\alpha_p$ 高 → 放大能力差异 → 好题 → 分子奖励
- $\lambda_p$ 高 → 结果反映真实能力 → 好题 → 分子奖励
- $|\gamma_p|$ 小 → 无系统偏移 → 好题 → 分母惩罚大偏移

**实验流程：**
1. Baseline：全部 100 题 + 3 个 judge 拟合 IRT → 基准排名和 ability
2. 对 K ∈ {20, 30, 40, 50, 60, 70, 80}：
   - IRT-filtered：取 quality_score 最高的 K 道题，重新拟合 IRT
   - Random baseline：随机选 K 道题，重复 10 次取均值

**评价指标：**
| 指标 | 含义 | 期望 |
|------|------|------|
| Spearman ρ | 筛选后排名与 baseline 排名的相关性 | 越接近 1 越好（排名保持） |
| Kendall τ | 同上，另一种排名相关性度量 | 越接近 1 越好 |
| Ability Std | 模型能力值的标准差 | 越大越好（区分度高） |
| Max Gap | 最强与最弱模型的能力差 | 越大越好（拉开差距） |

#### 4.3 Part 2：Rater 筛选 + 联合筛选

**Rater 质量评分公式：**

$$\text{quality\_score}_r = \frac{\alpha_r}{1 + |\beta_r|}$$

设计逻辑（与 Prompt 筛选对称，同样从 IRT 公式推导）：
- $\alpha_r$ 高 → 该 judge 能有效放大模型能力差异 → 好 judge → 分子奖励
- $|\beta_r|$ 小 → 该 judge 无系统性偏向 → 好 judge → 分母惩罚大偏差

**筛选策略（自适应）：**

| Rater 数量 | 策略 | 说明 |
|-----------|------|------|
| ≤ 4 个 | Leave-One-Out | 逐个去掉 1 个 rater，比较去掉后的评测效果，选出最差 rater |
| > 4 个 | Quality Score 排序 | 按 $\text{quality\_score}_r$ 从高到低排序，保留 top-N 个 rater |

阈值设为 4 的原因：LOO 的计算量为 O(n)，4 个 rater 只需 4 轮拟合，开销可接受；超过 4 个时若需保留 k 个，穷举组合 C(n, n-k) 会爆炸，直接用参数排序更高效。

**联合筛选流程：**
1. 用上述策略确定最优 rater 子集（去掉质量最差的 rater）
2. 在该 rater 子集的数据上重新拟合 IRT，获取新的 prompt 参数
3. 用新的 prompt quality score 做 Part 1 同样的 prompt 筛选实验
4. 验证两个维度联合筛选的叠加效果

---

## 四、预期结论

1. **IRT-filtered 优于 Random：** 在相同 K 下，IRT 筛选的 prompt 子集应在排名保持和区分度上显著优于随机选择
2. **少题高效：** 用 30-50 道高质量题目即可接近甚至超过 100 题全量的评测效果
3. **去噪增益：** 去掉低质量 judge 后，能力估计更准确，区分度更大
4. **联合筛选最优：** 同时筛 prompt 和 rater 的效果优于单独筛选

## 五、技术栈

| 组件 | 工具/框架 |
|------|----------|
| 模型推理 | vLLM |
| IRT 建模 | Pyro (概率编程) + PyTorch |
| 变分推断 | SVI + Trace_ELBO + Adam |
| 数据处理 | Pandas, NumPy |
| 统计检验 | SciPy (spearmanr, kendalltau) |
| 可视化 | Matplotlib |



结果整体符合预期，IRT 筛选在三个指标上都明显优于 Random。具体看：                                                                                                                                                            
                                                                                                                                                                                                                               
  Spearman ρ（左图）— IRT-filtered 几乎全程贴着 1.0，说明用少量高质量题就能保持排名不变。Random 在小 K 时只有 0.88 左右，差距明显。但 K=30 处有个异常下跌到 0.8，这个需要注意。                                                
                                                                                                                                                                                                                               
  Ability Std（中图）— IRT-filtered 从 K=30 开始就超过 baseline（蓝线 ≈ 2.0），说明去掉废题后区分度确实提升了。Random 要到 K=70-80 才追上 baseline。                                                                           
                                                                                                                                                                                                                               
  Max Gap（右图）— IRT-filtered 在 K=20-60 区间都高于 baseline（4.5-4.85 vs 4.5），Random 则一直低于 baseline 直到 K=80。这证实了去噪后能力差距被更充分地估出来。                                                              
                                                                                                                                                                                                                               
  一个问题：K=30 的 Spearman ρ 突然掉到 0.8                                                                                                                                                                                    
                                                            
  这是唯一不符合预期的点。K=20 排名完美，K=30 反而崩了，K=40 又恢复。可能的原因：                                                                                                                                              
                                                            
  - 这个实验用的是旧的 quality_score = α_p × λ_p，没有惩罚 γ_p。排名第 21-30 的 prompt 里可能混入了 |γ_p| 很大的题（高区分度但偏移严重），导致拟合出的排名被扭曲                                                               
  - 只有 4 个模型，Spearman ρ 对一次排名互换就很敏感（4 个模型交换两个，ρ 就能从 1.0 掉到 0.8）
                                                                                                                                                                                                                               
  v2 里加了 / (1 + |γ_p|) 的惩罚，这个 K=30 的异常大概率会被修复。跑一下 v2 对比看看就知道了。
